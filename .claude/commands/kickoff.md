# Kickoff

Full project kickoff — from "I have an idea" to "here's the plan, ready to build".

For a **feature on an existing project**, use `/explore` + `/plan` instead — they're faster and fit better into an ongoing session.

---

## When to use this

- Starting a brand new project
- A scope large enough that it deserves its own structured session before any code

---

## Phase 1 — Explore (open-ended, no agenda)

Open with:

> "Tell me about what you want to build. Doesn't need to be clear yet — just start talking."

Run the full `/explore` conversation from here:
- One question at a time, earned by what was just said
- Reflect back before asking the next thing
- Stay in problem space — no solutions, no tech yet
- End when the problem feels genuinely understood

Produce the explore summary before moving on.

---

## Phase 2 — Plan (structured, based on what emerged)

Pick up from the explore summary and run the full `/plan` sequence:

1. **Tech Lead** — research options, recommend reference architecture
2. **Designer** — flows, screens, components
3. **Architect** — data model, layers, security tier, cost
4. **QA** — edge cases, test strategy, definition of done

End with the written plan document and ask for approval.

**No code until approved.**

---

## After approval

The project exists but is empty. Suggest:

> "Plan approved — ready to build. Want to start with the first slice now, or run `/feature [first slice]` when you're ready?"
